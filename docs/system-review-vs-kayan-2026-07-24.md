# مراجعة نظامنا مقابل كيان — متحقَّقة من الكود الفعلي (2026-07-24)

مراجعة موحّدة عبر المودلات الثلاثة (حضور · أشخاص · رواتب) بعد الدراسة الحية الكاملة لكيان. **كل حالة
أدناه فُحصت في الكود الفعلي** (`grep`/قراءة) لا الذاكرة أو الوثائق — فبعض ما ظُنّ فجوة موجود، وبعض ما
ظُنّ موجوداً ناقص التطبيق.

## 0) تصحيحات مهمة كشفها الكود (الوثائق كانت مضلِّلة)

| البند | ما قالته الدراسة | **الواقع بالكود** |
|---|---|---|
| أثر القاعدة المالي | «فجوة كبرى» | ✅ **مبنيّ**: `ShiftRuleStore.ActionTypes` = **٧ أنواع** (مخالفة/ملاحظة/إجازة/مغادرة/أوفرتايم/دخل/اقتطاع، أكثر من كيان الخمسة) + `AttendanceTransactionStore` + جسر الترحيل للمسير |
| الحقول المخصصة | «محرّك ديناميكي غائب» | ✅ **محرّك حقيقي**: `EntityCustomFields` (HrEntityFieldDefs/Values) عبر **١٢ كياناً** (عائلة/عقود/علاوات/تعليم/خبرات/شهادات/تدريب/طبية/عهد/عنوان/طوارئ/إقامة) بـ**٦ أنواع** (text/number/date/textarea/select/checkbox) |
| محرّك التقارير | «غائب» | ◐ **موجود لكن للأشخاص فقط**: `PeopleReportsStore` + `PeopleReportCatalog`. الفجوة = **تعميمه** (مصادر حضور/رواتب + محور) لا بناؤه من الصفر |
| تسلسل الإجراءات التصاعدي | «مبنيّ» | ⚠️ **مخزَّن لا مطبَّق**: عمود `UseEscalation` موجود بـ`ShiftRuleStore` لكن **لا تطبيق له** في `RecommendationStore` |

## 1) 🔴 الفجوات المانعة / الأعلى قيمة (مؤكَّدة بالكود)

### 1.أ محرّك المعادلات — **لا مُقيّم إطلاقاً** (#1)
- بحث الكود: **لا `FormulaEvaluator`/`NCalc`/`DataTable.Compute`/`Evaluate` في أي مكان**.
- `SalaryItemStore` يخزّن عمود `Formula` (بذرة UI) لكن `PayrollRunStore` **لا يقرأه ولا يقيّمه** (صفر مطابقة
  لـFormula" في المسير).
- **الأثر**: عند كيان الصيغة هي العمود الفقري (عناصر + **قاعدة الضريبة + قاعدة الضمان + الإعفاءات كلها صيغ**).
  عندنا القيم ثابتة/مبرمجة صلباً. **بناء مُقيّم صيغ آمن وربطه بالمسير = أعلى نقلة، ويفتح التوطين بلا كود.**

### 1.ب محرّك التقارير العام (تعميم القائم)
- عندنا `PeopleReportsStore` (باني تقارير أشخاص). الناقص: **مصادر بيانات الحضور والرواتب** + **الوضع المحوري
  (Pivot)** + مشاركة ESS. البنية موجودة — التعميم أرخص من البناء.

### 1.ج ربط الإجازات/المغادرات بالمسير
- عندنا `/LeaveBalances` + `/LeaveRequests` منفصلة عن الرواتب. الناقص: **كتالوج أنواع الإجازة للرواتب**
  (مدفوعة/بلا رصيد/غير مدفوعة/مغادرات + أعلام «خصم من الخدمة») + **بدل الإجازات + أثر المغادرات المالي**.

## 2) 🟠 فجوات وظيفية كبيرة (مؤكَّدة غائبة بالكود)

| # | الفجوة | الحالة بالكود |
|---|---|---|
| 2.أ | **تطبيق التسلسل التصاعدي** للإجراء التأديبي | العمود موجود، التطبيق مفقود بالمقيّم |
| 2.ب | **سعر ساعة العمل بالأيام غير-العادية** (معامل أوفرتايم لعطلة/راحة/عطلة رسمية/إجازة) | ✗ لا حقول بـ`ShiftTypeStore` |
| 2.ج | **سياسة الحضور العامة** (سقوف عطلة/راحة · سقف/طلب البصمات المفقودة · إعادة تحليل تلقائية عند الموافقات · إنفاذ geofence · ضوابط بصم الويب) | ✗ لا مطابقة |
| 2.د | **قواعد أسبوعية/شهرية بشرائح تصاعدية** (`SliceFrom/SliceTo`) | ✗ عندنا يومية بشرط مفرد |

| 2.ح | **محرّك العمليات/المهام** (onboarding بقوالب + حقول لكل مهمة + تتابع) | ◐ `/EmployeeTasks` أبسط |
| 2.ط | **نظام المحاكاة** (what-if) · **القروض** (أقساط) · **الاحتياطي** (مخصصات) | ✗ لا Store لأيٍّ منها |
| 2.ي | **قوالب القسائم المتعددة** + **السنة المالية** (رواتب إضافية 13/14) + **سقوف الاقتطاع** | ✗ قسيمة واحدة ثابتة، لا سقوف |

## 3) 🟡 مولّدات دمج + مودلات كاملة (قرار منتج)
- **مصمّم البطاقات SVG** + **توليد الوثائق بقوالب دمج** — محرّك دمج موحّد (مع باني النماذج + محرّك المعادلات).
- **تقييم الأداء** · **إدارة العقود كشاشة (عدّاد+توليد)** · **الإقرارات** · **رئيس وحدة مؤقت** · **تاريخ إسقاط
  الإجراء التأديبي** · **ملفات بنوك قابلة للتهيئة** (نتفوّق بامتلاك CSV، ننقصه المرونة) · **ربط ERP** · **توزيع
  التكلفة**.

## 4) ✅ ما نتفوّق فيه أو حقّقنا باريتي (متحقَّق)
- **سياسة السماحية داينمك** (`GraceExceededPolicy` — بُني اليوم) · **بوابة الاعتماد الشهري** (`ApproveWithGateAsync`)
  · **تصدير ملف البنك** (`BankFileRowsAsync`) · **محرّك الهوية White-Label** · **لوحة الأشخاص القابلة للتخصيص**.
- **باريتي**: دورة المسير (4 حالات) · GOSI/ضريبة/EOS تعريفاً · حركات الدخل/الاقتطاع/الأوفرتايم/الزيادات/نهاية
  الخدمة · تحديثات الموظف (7 تصنيفات+قفل) · الملف 360° · **محرّك الحقول المخصصة** (١٢ كياناً) · جسر الحضور←المسير.

## 5) ترتيب البناء المقترح (بالأثر/الكلفة)

1. **مُقيّم الصيغ + ربطه بالمسير** (1.أ) — يفتح التوطين والمرونة كلها؛ نبني على بذرة `SalaryItemStore.Formula`.
2. **تطبيق التسلسل التصاعدي** (2.أ) — عمود موجود، يلزم منطق التقييم فقط (كلفة صغيرة، أثر تأديبي حقيقي).
3. **تعميم محرّك التقارير** (1.ب) — نضيف مصادر حضور/رواتب + محور لـ`PeopleReportsStore` القائم.
4. **سعر ساعة العمل بالأيام غير-العادية** (2.ب) — حقول بسيطة بـ`ShiftTypeStore` تفعّل أجر الأوفرتايم الصحيح.
5. **ربط الإجازات/المغادرات بالمسير** (1.ج) + **سياسة الحضور العامة** (2.ج).
6. ما بعده حسب قرار المنتج: باني النماذج · الاستبيانات · القروض/الاحتياطي/المحاكاة · قوالب القسائم · الهياكل الثلاث.

> **الخيط الجامع**: بنودٌ كثيرة (المعادلات · باني النماذج · الحقول المخصصة · البطاقات/الوثائق · الاستبيانات)
> هي وجوهٌ لـ**محرّك حقول/تعابير/دمج ديناميكي واحد**. الاستثمار فيه مرّة يخدم المودلات الثلاثة — وهو بالضبط
> ما يميّز كيان. عندنا نصفه مبنيّ (`EntityCustomFields`) وينقصه المُقيّم والباني والدمج.
