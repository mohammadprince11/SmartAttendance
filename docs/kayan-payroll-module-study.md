## 1) شجرة مودل الرواتب /Payroll

### أ. الاحتساب والتنفيذ (المسير)
| الشاشة | الرابط | الدور |
|---|---|---|
| **حساب الرواتب** | `/MonthlySalaryCalculation/Index?Type=1` | المسير الشهري ⭐ |
| الرواتب الإضافية | `?Type=2` | مسير إضافي (بونص/13th month) |
| حساب الاحتياطي | `/ProvisionCalculation` | مخصصات محاسبية (مكافأة/إجازات) |
| احتساب خارج الرواتب | `/OutofPayrollCalculation` | دفعات خارج المسير |
| إنهاء الموظف | `/EmployeeTermination_IndexFromPayroll` | نفس خط أنابيب الأشخاص (تسوية) |
| **تصدير ملف البنك** | `/ExportBankReport` | ملفات التحويل البنكية |
| **ربط النظام المالي** | `/MonthlyErpIntegration` | تكامل ERP (قيود محاسبية) |
| حساب توزيع التكلفة | `/CostDistribution` | توزيع على مراكز التكلفة |
| إعادة حساب راتب الضمان | `/CalculateSocialSecuritySalary` | إعادة احتساب وعاء الضمان |
| مدقق حركات كيان | `/KayanSimulator/...TransactionsAuditor` | **محاكي/مدقق قبل التنفيذ** |

### ب. الحركات (Transactions — نفس نمط دفتر الحركات بالأشخاص)
دخل `/IncomeAndDeduction?Type=1` · اقتطاع `?Type=2` · عمل إضافي `/Overtime` · تعديل أيام الراتب `/SalaryDaysDeduction` · زيادة الراتب `/Raises` · تحديثات مكافأة نهاية الخدمة `/STBUpdates` · إجازات `/Leaves` · مغادرات `/Permissions` · تدوير المغادرات `/RoundingManagement` · بدل إجازات `/LeaveEncashment` · تعديل إجازات `/LeaveAdjustment` · أرصدة الإجازات `/BalanceManagement` · عناصر خارجية `/ExternalItemsManagement` · ساعات دوام جزئي `/PartTimersWorkingHours` · **مستورد الحركات** `/TransactionsImporter` · حد العمل الإضافي `/MaximumOvertimeManagement` · قروض الموظفين `/Loans`.

### ج. الإعدادات
**عناصر الراتب** `/SalaryItems?PageType=…` — **10 أنواع**: الدخل (1) · الاقتطاعات (2) · العمل الإضافي (3) · الدخل القائم على الحصص (4) · العناصر الخارجية (5) · إدخال البيانات (6) · الامتيازات الخارجية (8) · تعديل أيام الراتب (9) · القيم الثابتة (10) · الامتيازات السنوية (11).
**التصنيفات** `/SalaryClassifications`: فئات التأمين الصحي · التأمين الصحي · الضمان الاجتماعي · الضمان الإضافي · مكافأة نهاية الخدمة.
**الضريبة** `/TaxClassifications`: إعفاءات · إعفاءات إضافية · ملفات الضريبة · ملفات إضافية · إجمالي الراتب · ضريبة الحد الأدنى.
**سلم الرواتب** `/SalaryScale`: **ثلاثة أبعاد** (البعد 1/2/3) + فئات + مجموعات + تحديثات السلم `/SalaryScaleUpdates`.
**البنوك** `/Banks`: البنوك · الفروع · بنوك الشركة · **ملفات البنوك** (صيَغ التصدير) · **منشئ ملخص البنك** · **عناصر محسوبة**.
**أخرى**: التهيئة `/Configuration` · السنة المالية `/FinancialYear` · إعدادات القرض `/Loan` · مركز إشعارات الرواتب · سجل تدقيق.

> **ملاحظات معمارية أولية**: (1) نفس نمط دفتر الحركات + القفل بكل حركات الرواتب؛ (2) الإنهاء يظهر بالمودلين (تكامل التسوية)؛ (3) «مدقق الحركات» = محاكاة قبل الاعتماد؛ (4) سلم رواتب ثلاثي الأبعاد؛ (5) ملفات بنوك قابلة للبناء (منشئ ملخص + عناصر محسوبة) لصيَغ التحويل المختلفة.

## 2) حساب الرواتب (المسير) — فحص حي

**قائمة المسيرات**: رقم الدفعة (نمط `2026-8-1` سنة-شهر-تسلسل) · نوع الراتب · الفترة · **حوسب بواسطة + طابع زمني** · عدد الموظفين · مجموع الرواتب المدفوعة · **مقفل** · **تم إصدارها** · **تم إرسال القسيمة**.
**→ دورة حياة المسير**: حوسِب ← قُفل ← أُصدر ← أُرسلت القسائم — 3 بوابات منفصلة بعد الاحتساب. مثال حي: دفعة 8-2026 بـ874 موظفاً بمجموع 13.5M JOD.
مجموعات التنقل بالمودل: حركات · العمليات المالية · حركات الراتب · **استعلام القسائم** · إدارة القروض · إدارة سلم الرواتب · **نظام المحاكاة** · تقارير · إعدادات.

## 3) نموذج عنصر الراتب (الدخل) — فحص حي ⭐

| المجموعة | الحقول |
|---|---|
| الهوية | اسم الدخل (عربية/لغة أخرى + English) · **النوع: علاوة / حركة دخل** (العلاوة = متكررة وتظهر بملف الموظف المالي — لوحة العلاوات اللي طبقناها!) |
| النطاق | **داخل الراتب / خارج الراتب** |
| القيمة | **ثابت / المعادلة** (محرك معادلات!) + القيمة الافتراضية |
| الضريبة | خاضع / غير خاضع · خصم الضريبة من الحركات خارج الراتب |
| الاحتساب | يشمل الراتب الإضافي · **الأجر الشامل** · **نسبي** (prorated) |
| التوقيت | تاريخ الحركة / الشهر السابق / الشهر اللاحق (×2: بداية/نهاية؟) |
| عامة | العملة · رقم الوحدة · نشط · ملاحظة |
| **الخدمة الذاتية** | إمكانية الطلب من ESS: الموظف / المدراء / كلاهما + **المرفق مطلوب** + **السبب مطلوب** |
| التكامل | ربط بحقول إضافية (معلومات الحساب البنكي / الحقول المخصّصة) |

→ العنصر الواحد يحمل: تعريفه المالي + قاعدته الضريبية + سلوكه بالاحتساب + صلاحية طلبه من ESS. هذا «عنصر الراتب» الذي تختار منه لوحة العلاوات بملف الموظف.

## 4) خريطة تأثير مودل الرواتب (أولية)
| المكوّن | يقرأ من | يؤثر على |
|---|---|---|
| المسير الشهري | حركات (دخل/اقتطاع/إضافي/إجازات/قروض) + المعلومات المالية للموظف + الحضور | القسائم · ملف البنك · ERP · **قفل حركات الأشخاص** |
| عناصر الراتب | — (تعريفات) | العلاوات بملف الموظف · الحركات · ESS (طلبات) · الضريبة |
| أسباب الإيقاف (STB) | مشترك مع الأشخاص | نسبة مكافأة نهاية الخدمة بالتسوية |
| حركات الإجازات/المغادرات | مودل الإجازات والحضور | خصومات/بدلات المسير |
| القروض | إعدادات القرض | أقساط شهرية بالمسير |
| سلم الرواتب (3 أبعاد) | — | راتب الموظف عند الربط بالسلم + تحديثات السلم الجماعية |
| المحاكاة/المدقق | كل ما سبق | فحص قبل الاعتماد |

## 5) فجوات SmartAttendance الكبرى (أولية)
عندنا لا يوجد مودل رواتب بعد — الدراسة تعطي خارطة الطريق: نبدأ بـ(1) عناصر راتب (عندنا lookup العلاوات — نرقّيه لكيان كامل بنوع/قيمة/ضريبة)، (2) مسير شهري بدورة حياة (حوسب/مقفل/مُصدر)، (3) حركات دخل واقتطاع، (4) قسيمة راتب. المعادلات والضريبة والضمان مراحل لاحقة.
