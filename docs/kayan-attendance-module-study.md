## 2) منشئ قواعد المناوبات ⭐ — محرك «حضور ← مخالفة» (فحص حي)

**القائمة**: قواعد بصيغة «في حالة [نوع بصمة] [حدث] [مشغّل + قيمة]، الإجراء [إجراء تأديبي]» + مفتاح **«اتخذ الإجراء تلقائياً»** (نعم/لا).
أمثلة حية: «استراحة عدد المرات أكثر من 3 → سلوك سيء - انذار شفوي (تلقائي)» · «الحضور المدة أقل من 2 ساعات → إجراء (تلقائي)» · «ختم الدخول بعد 01:00 AM → ...».

**نموذج القاعدة** (أبعاد الاستهداف × الشرط × الإجراء):
| البعد | الخيارات |
|---|---|
| المناوبات | multi-select من المناوبات المعرّفة + اختيار الكل |
| نطاق اليوم | أيام العمل · يوم عطلة · يوم راحة · يوم راحة (عطلة رسمية) · العطل · إجازة · **رحلات العمل** |
| أيام الأسبوع | multi (سبت–جمعة) |
| **نوع البصمة** | الحضور · استراحة · أخرى · مهمة عمل · اذن شخصي · داخلي · **صلاة** · **دخان** (من «الدلالات المخصصة» — أنواع بصمات قابلة للتعريف!) |
| **الحدث** | ختم الدخول · ختم الخروج · المدة · غائب · ختم دخول مفقود · ختم خروج مفقود · Attendance Timeline · ساعات التأخير · ساعات الخروج المبكر |
| المشغّل | قبل · قبل/في · بعد · بعد/في · بين · على (+ قيمة وقت/مدة/عدد مرات) |
| الإجراء | من قائمة الإجراءات التأديبية (مودل الأشخاص!) + «اتخذ تلقائياً» |

**→ الربط الأخطر بالنظام**: قاعدة حضور ⟶ تولّد **مخالفة/إجراء تأديبي** تلقائياً أو كاقتراح (شاشة «متابعة الإجراءات المقترحة») ⟶ الإجراء يخصم بالرواتب. الحضور والمخالفات والرواتب سلسلة واحدة.

## 3) إعدادات تسجيل الحضور (فحص حي)

### 3.1 التهيئة `?PageType=5` — مفاتيح المودل
- **الحقول الإلزامية**: نوع الدوام إلزامي (منتظم / مجدول / لا شيء) + مناوبة افتراضية.
- **تعريف الفترات**: الشهري (آخر يوم = تاريخ تقويم / تاريخ محدد — يدعم شهر رواتب مخصص!) · الأسبوعي (أسبوع تقويمي / أسبوع عمل + يوم البداية).
- **حدود الدوام المجدول**: حد أقصى لأيام العطلة وأيام الراحة (يوم/فترة أسبوعية أو شهرية).
- **طلبات البصمات المفقودة**: حد شهري (مرات) + نافذة زمنية للطلب (أيام) + سبب إلزامي.
- **إعادة التحليل** ⭐: «إعادة تحليل بيانات الحضور اليومي تلقائياً بمجرد الموافقة على طلبات المغادرة والإجازة والبصمات المفقودة» + **«منع إعادة التحليل إذا اتُّخذت إجراءات على حضور الموظف»** (حماية القرارات المتخذة).
- بصمات: أول دخول/آخر خروج · السماح للمجدول بالبصم بلا جدول · **أنواع الدلالات المسموحة عند البصم** (7 مختارة) + **المسموح طلبها بالبصمة المفقودة** (استراحة، مهمة عمل).
- خدمة ذاتية وموبايل: إخفاء التأخير من «حضوري السابق» · منع البصم عبر الويب · السماح بالدخول/الخروج **خارج المنطقة الجغرافية** (مفتاحان منفصلان geo-fence).

### 3.2 تهيئة الدلالات `?PageType=1`
جدول ثنائي اللغة (English/العربية) لأنواع البصمات المخصصة — يغذّي: أزرار البصم بالخدمة الذاتية، منشئ القواعد (نوع البصمة)، وطلبات البصمة المفقودة. أمثلة حية: صلاة، دخان، مهمة عمل، اذن شخصي، داخلي.

### 3.3 مصدر بيانات الحضور `?PageType=2`
قائمة مصادر (اسم / قراءة من / دلالات نعم-لا / حالة). **أنواع القراءة**: **ملف إكسل** · **اسم عرض البيانات/الجدول** (سحب مباشر من قاعدة بيانات جهاز البصمة!) · **Kayan External API**. → التكامل مع الأجهزة عبر DB view أو API، مو driver لكل جهاز.

### 3.4 مستعرض الحضور `/AttendanceViewer`
مصفوفة **موظف × أيام الشهر** بمفتاح ألوان: حاضر · غائب · في إجازة · عطلة نهاية أسبوع · يوم راحة · عطلة رسمية · يوم راحة (عطلة رسمية) · **في رحلة عمل** + عمود «المجموع x/31». مع «إعدادات عارض مسبقة» (Views محفوظة).

## 4) مودل «سجل الدوام» TimeSheet — ساعات المشاريع (مختلف عن الحضور!)
جرد: نوع سجل دوام الموظف · **تعيين الموظف على مشروع** · إدارة السجل اليومي/الشهري · إعدادات: **العملاء · المشاريع · مهام · العلامات** · تقارير وإشعارات.

## 5) خريطة تأثير مودل الحضور
| المكوّن | يقرأ من | يؤثر على |
|---|---|---|
| منشئ قواعد المناوبات | المناوبات + الدلالات + الإجراءات التأديبية (أشخاص) | **مخالفات تلقائية/مقترحة** ← خصومات الرواتب |
| التهيئة (إعادة التحليل) | موافقات الإجازات والبصمات | إعادة احتساب الحضور اليومي تلقائياً |
| الدلالات المخصصة | — | أزرار البصم بالـ ESS + قواعد الاحتساب + طلبات التصحيح |
| مصادر البيانات | أجهزة البصمة (DB/Excel/API) | البصمات الخام ← التحليل اليومي |
| المواقع الجغرافية | تعريف Geo + ربط الموظفين | صلاحية البصم بالموبايل (geo-fence) |
| نمط الحضور (منتظم/مجدول/بلا) | كيان الموظف (معلومات الحضور) | القواعد المطبقة + فترة الإنذار (أشخاص) + العارض |
| طلبات البصمات المفقودة | حدود التهيئة + الدلالات المسموحة | الموافقات ← إعادة التحليل ← الحضور |
| الحضور اليومي/الأسبوعي/الشهري | كل ما سبق | **مسير الرواتب** (ساعات، تأخير, غياب) + تقارير |

## 9) الحضور اليومي — الشاشة التشغيلية الأم (فحص حي)
- **أزرار المحرك أعلى الشاشة**: «استيراد الحضور» ← «تحديث الحضور» ← «تحليل واسترجاع الإقتراحات» (خط الأنابيب: بصمات خام ← تحليل ← اقتراحات إجراءات).
- **عرضان**: «يوم» (شبكة أيام الشهر) و«الموظفين» (جدول). فلاتر: الفترة (شهر، آخر 3 أشهر متاحة)، وحدة عمل (22)، الهيكلية (23)، الكفيل، المنصب (898)، نوع العقد، بحث اسم/رمز.
- **جدول الموظفين**: رمز · اسم · وحدة عمل · الهيكلية · نوع الدوام · **حالة الحضور = "Under review"** · الحالة (نشط) ← **الشهر له دورة حالة مراجعة لكل موظف** (تُعتمد ثم تُقفل للرواتب).
- **شاشة شهر الموظف** (بالنقر على الصف): هيدر «حضور - 7/2026» بأزرار **حذف الحضور** و**تحليل واسترجاع الإقتراحات**. 4 تبويبات:
  1. **حضور وغياب**: جدول اليوميات — التاريخ · مناوبة · ختم الدخول · ختم الخروج · الفترة · **ساعات التأخير** · **ساعات الخروج المبكر** · الحالة · **تم التحليل** (علم لكل يوم!) + «إضافة سجل الحضور».
  2. **البصمات الأخرى** (الخام غير المصنفة).
  3. **الإجراءات المقترحة (n)** — مخرجات المحرك بانتظار الفرز.
  4. **الحركات (n)** — الحركات المولّدة (إجازة/مخالفة/أوفرتايم...).
- **درج «بحث متقدم»**: لاقط موظفين هرمي (رمز/وحدات عمل بشجرة اختيار) + **«الموظفين من جدول البيانات»: رفع/لصق ملف Excel لتحديد الموظفين** (EmployeePickerSpreadSheet — فكرة تستحق النسخ).

## 10) منشئ قواعد المناوبات ⭐⭐ — المخطط الكامل (فحص نموذج «قاعدة جديدة»)
ثلاثة منشئات (يومي/أسبوعي/شهري) بنفس النموذج. عرض القاعدة بالقائمة: «في حالة [نوع بصمة] [دلالة] [شرط] [قيمة], الإجراء [أثر]» + «اتخذ الإجراء تلقائياً: نعم/لا».

**أمثلة حية بالديمو (يومي)**: استراحة عدد المرات أكثر من 3 ← مخالفة «سلوك سيء - إنذار شفوي» (تلقائي) · الحضور ختم الدخول قبل/في 09:00 ← بلا إجراء (رصد) · الحضور المدة أقل من 2 ساعات ← تلقائي · الحضور ختم الدخول بعد 01:00 ← تلقائي · الحضور ختم الدخول قبل/في وقت بدء المناوبة ← مخالفة.

**بنية نموذج القاعدة** (3 تبويبات: القواعد · معايير الاستحقاق · المناوبات):
- **النطاق**: المناوبات (multiselect بالمناوبات المعرفة: صباحي/طبيعي/وردية طبيعية...) + «طبق هذه القاعدة على» `ApplyOnlyIn`: **أيام العمل / يوم عطلة / يوم راحة / يوم راحة (عطلة رسمية) / العطل / إجازة / رحلات العمل** + أيام الأسبوع (multiselect 7).
- **الشرط «في حالة»**:
  - نوع البصمة `PunchTypeID` (9 أنواع semantic معرفة: الحضور، استراحة، أخرى، مهمة عمل، إذن شخصي، داخلي، صلاة، دخان — قابلة للتوسيع من «دلالات مخصصة»).
  - **الدلالة** `FunctionTypeID` (9): ختم الدخول · ختم الخروج · **المدة** · **غائب** · ختم الدخول مفقود · ختم الخروج مفقود · Attendance Timeline · **ساعات التأخير** · **ساعات الخروج المبكر**.
  - **الشرط** `SignID` (6): قبل · قبل/في · بعد · بعد/في · **بين** · على.
  - **القيمة الزمنية**: مطلقة HH:MM **أو نسبية**: «وقت بدء المناوبة» / «وقت انتهاء المناوبة»، ولنطاق «بين»: من/إلى، وكل طرف له **مرساة يوم**: من اليوم السابق / نفس اليوم / اليوم التالي (حل عبور منتصف الليل!).
- **الإجراء**:
  - «استخدام تسلسل الإجراءات» (checkbox — سلسلة إجراءات متدرجة بالتكرار مثل لائحة المخالفات).
  - **نوع الإجراء** `ActionTypeID` (6): **الإجازات · المغادرات · العمل الإضافي · الدخل · اقتطاع · مخالفة** ← المحرك يضرب مباشرة بالرواتب (دخل/اقتطاع) والإجازات والأوفرتايم، مو بس المخالفات!
  - الإجراء المحدد `PolicyActionID` (يعتمد على النوع — قائمة تُملأ حسب الاختيار).
  - «اتخذ الإجراء تلقائياً» + **«السماح على تعديل الإجراء الموصى به»** (يتحكم بمرونة شاشة فرز الاقتراحات).

**الاستنتاج المعماري**: القاعدة = (نطاق: مناوبات × سياق يوم × أيام أسبوع × معايير استحقاق) + (شرط: نوع بصمة × دلالة × مقارنة × قيمة زمنية نسبية/مطلقة بمرساة يوم) + (أثر: نوع × إجراء × تلقائي؟ × قابل للتعديل؟ × تسلسل تصاعدي؟). التحليل يمر على بصمات كل يوم فيولّد «إجراءات مقترحة» تُفرز بشاشة مستقلة أو تُتخذ تلقائياً فتصير «حركات».

## 11) تهيئة المناوبات — تعريف المناوبة (نموذج «مناوبة عمل جديدة»)
- القائمة: اسم المناوبة · نوع الدوام (**ثابت/مرنة**) · أيام العطل (لكل مناوبة أيامها!).
- النموذج بتبويبين: **تفاصيل** + **الفترات** («قسّم المناوبة إلى فترات متعددة، لكل منها بداية ونهاية» — سبليت شفت).
- تبويب التفاصيل: اسم ثنائي اللغة + **لون المناوبة** (8 ألوان — للمستعرض) + فترة واحدة/فترات متعددة + ثابت/مرنة + **مصفوفة 7 أيام** `lstFixedShiftDetails[0..6]`: لكل يوم أسبوع — نوعه (يوم عمل/يوم عطلة/يوم راحة) + ساعات (ShiftD) + دقائق (ShiftM) + وقت الدخول (CheckIn) + وقت الخروج (CheckOut). أي **المناوبة الثابتة تُعرّف يوماً-بيوم**، والمرنة بعدد ساعات مطلوب.

## 12) التهيئة العامة (PageType=5) — كل المفاتيح الملتقطة
- **الحقول الإلزامية**: «نوع الدوام حقل إلزامي» + نوع الدوام (دوام منتظم/دوام مجدول/لا شيء) + **مناوبة افتراضية** (طبيعي).
- **الحضور الشهري**: «اعتبر آخر يوم بالشهر»: تاريخ التقويم / **تاريخ محدد** (يقفل دورة الشهر على يوم قطع مخصص للرواتب).
- **الأسبوعي**: بداية الأسبوع: أسبوع تقويمي/أسبوع عمل + اليوم.
- **سقوف المجدول**: حد أقصى لأيام العطلة/الراحة بالدوام المجدول (يوماً لكل أسبوع/شهر).
- **البصمة المفقودة**: حد أعلى شهري (مرات) + نافذة السماح بالطلب (أيام) + «سبب البصمة المفقودة إلزامي» + **الدلالات المسموح طلبها بالبصمة المفقودة** (استراحة، مهمة عمل).
- **إعادة التحليل**: «إعادة تحليل بيانات الحضور اليومي تلقائياً بمجرد الموافقة على المغادرة/الإجازة/البصمات المفقودة» + **«منع إعادة تحليل الحضور إذا اتُّخذت إجراءات على حضور الموظف»** (حماية القرارات المتخذة).
- **البصم**: «السماح لموظفي المجدول بالبصمة بلا جدول» + **«أول بصمة دخول وآخر بصمة خروج»** (نمط الالتقاط) + «الدلالات المسموح اختيارها عند تسجيل البصمة» (7 مختارة) + «عدم السماح بالبصم عبر الويب» + سماحان منفصلان للبصم **خارج المنطقة الجغرافية** (دخول/خروج).
- **ESS**: «إخفاء التأخير والخروج المبكر من (حضوري السابق) بالخدمة الذاتية».

## 13) بقية الشاشات (لقطات سريعة)
- **تهيئة الدلالات (PageType=1)**: جدول بسيط (الدلالة EN/AR) — به عُرّفت الأنواع التسعة (صلاة/دخان/مهمة عمل...).
- **مصدر بيانات الحضور (PageType=2)**: قائمة مصادر (الديمو: Excel). النموذج: اسم + «قراءة من» `IntegrationTypeID`: **ملف إكسل / اسم عرض البيانات-الجدول (DB مباشر من جهاز البصمة) / Kayan External API** + علم دلالات + نشط.
- **متابعة الإجراءات المقترحة**: تبويبان (الإجراءات المقترحة/الحركات) + 3 عدادات: مقترحة · **حركات متضاربة** · **متجاهلة**. أعمدة: موظف · مناوبة · تاريخ · نوع بصمة · دخول · خروج · فترة · **الإجراء المقترح** (+ اعتماد/تجاهل).
- **الحضور الشهري**: بطاقة لكل شهر بحالة «تحليل الحضور اليومي غير مكتمل» / «لا يوجد بيانات» — **التجميع الشهري مشروط باكتمال تحليل اليومي** (وهو بوابة الرواتب).
- **مناوبات العمل الثابتة**: جدول موظفين (رمز/اسم/وحدة/هيكلية/موقع/**تهيئة المناوبات**) + «تحديد مناوبات الموظفين» (تعيين جماعي).
- **جدولة مناوبات العمل (Roster)**: شبكة موظف × كل أيام الشهر + زر **«نشر الجدول»** + عرض حسب الموظفين/المناوبات + fallback تلقائي للمناوبة الافتراضية لأي يوم غير مجدول.
- **مستعرض الحضور**: شبكة موظف × أيام الشهر بمفتاح 8 حالات ملونة: حاضر · غائب · في إجازة · عطلة نهاية أسبوع · يوم راحة · عطلة رسمية · راحة (عطلة رسمية) · **في رحلة عمل** + عمود مجموع (n/31). وإعدادات عارض مسبقة (AttendanceViewerSetup).
- **تعديل مناوبات مؤقت / بصمات أونلاين / جيو / عمل من المنزل**: شاشات مساندة (تجاوز مناوبة لفترة، بصمات الموبايل/الويب المباشرة، أسوار جغرافية، وWFH كنوع يوم).

## 14) الخلاصة المعمارية النهائية — نموذج البيانات المستنتج
```
ShiftType (مناوبة): اسم ثنائي، لون، ثابت/مرن، فترة/فترات، Days[7]{نوع اليوم، ساعات، دخول، خروج}
PunchSemantic (دلالة): اسم ثنائي — تصنيف البصمات (حضور/استراحة/صلاة/...)
AttendanceSource: اسم، نوع التكامل (Excel/DBView/API)، مفعّل
EmployeeShift: موظف ← مناوبة ثابتة | جدول شهري (RosterCell: موظف×يوم←مناوبة/عطلة/راحة) | تجاوز مؤقت
RawPunch (بصمة خام): موظف، وقت، دلالة، مصدر، موقع جغرافي؟
DayAttendance (يومية): موظف×تاريخ — مناوبة، دخول/خروج، فترة، ساعات تأخير، ساعات خروج مبكر، حالة، تم التحليل
ShiftRule (قاعدة): منشئ (يومي/أسبوعي/شهري)، مناوبات[]، سياق يوم (7 قيم)، أيام أسبوع[]، معايير استحقاق،
  شرط {نوع بصمة، دلالة (9)، مقارنة (6)، قيمة زمنية (مطلقة/نسبية لبداية-نهاية المناوبة) بمرساة يوم (سابق/نفسه/تالي)}،
  أثر {نوع (إجازة/مغادرة/أوفرتايم/دخل/اقتطاع/مخالفة)، إجراء، تلقائي؟، قابل للتعديل؟، تسلسل؟}
RecommendedAction (اقتراح): يومية×قاعدة ← إجراء مقترح، حالة (معلق/معتمد/متجاهل/متضارب)
AttendanceTransaction (حركة): الأثر المنفذ — يرتبط بالمخالفات/الإجازات/الرواتب
MonthAttendance (شهر الموظف): حالة (Under review ← معتمد/مقفل) — بوابة الرواتب
```
**خط الأنابيب**: استيراد بصمات ← تصنيف بالدلالات ← «تحديث الحضور» (مطابقة مناوبة/يوم) ← «تحليل واسترجاع الاقتراحات» (تشغيل القواعد) ← فرز الاقتراحات (أو تلقائي) ← حركات ← اكتمال تحليل الشهر ← اعتماد شهري ← رواتب. الموافقات (إجازة/مغادرة/بصمة مفقودة) تعيد التحليل تلقائياً ما لم تُتخذ إجراءات.

## 15) خريطة البناء المقترحة لنظامنا (عند فك تأجيل الحضور)
1. **ShiftTypes** (تعريف يوم-بيوم + مرن + فترات) — يستبدل Shifts الحالي البسيط.
2. **PunchSemantics + AttendanceSources** (الاستيراد الحالي يتطور لمصادر معرفة).
3. **إعادة بناء سجل اليومية** DayAttendance بالحقول المشتقة (تأخير/خروج مبكر/تم التحليل).
4. **محرك القواعد** ShiftRules + التحليل + الاقتراحات + شاشة الفرز — الأثر يرتبط بمخالفاتنا (موجودة) وإجازاتنا (موجودة) والأوفرتايم/دخل/اقتطاع (مع الرواتب).
5. **Roster** شهري بالنشر + المستعرض الملون.
6. حالة شهرية للموظف Under review ← اعتماد ← (لاحقاً) قفل رواتب.
